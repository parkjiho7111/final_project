🖱️ [팀 가이드] 마우스로 끝내는 Git 브랜치 (VS Code 편)
명령어 외우기가 어렵다면 이 방법을 쓰세요. 터미널 명령어와 100% 똑같은 기능입니다.

1단계: 브랜치 만들고 이동하기
가장 중요한 단계입니다. 현재 내가 어디서 작업하는지 확인해야 합니다.

VS Code 가장 왼쪽 하단에 있는 main (또는 master) 글씨를 클릭하세요.

화면 위쪽에 입력창이 뜹니다. 여기서 **Create new branch... (새 분기 만들기...)**를 선택합니다.

새로운 브랜치 이름(예: feature/login)을 적고 엔터(Enter)를 칩니다.

왼쪽 하단 글씨가 feature/login으로 바뀐 것을 확인합니다. (이제 이동 완료!)

2단계: 작업 저장하기 (Add & Commit)
코드를 수정하면 Source Control 아이콘에 숫자(변경된 파일 개수)가 뜹니다.

Source Control 탭을 누릅니다.

Add (git add): 변경된 파일 목록 옆에 있는 + (플러스) 버튼을 누릅니다. (파일이 'Staged Changes'로 올라갑니다.)

Commit (git commit): 위쪽 메시지 입력칸에 "로그인 기능 구현" 같은 설명을 적습니다.

Commit 버튼: 입력칸 바로 위에 있는 'Commit(커밋)' 버튼(혹은 체크 표시 ✔)을 누릅니다.

3단계: 깃허브에 올리기 (Push)
내 컴퓨터에 저장된 내용을 깃허브 웹사이트로 보냅니다.

커밋을 하고 나면 'Publish Branch(분기 게시)' 또는 **'Sync Changes(변경 내용 동기화)'**라는 파란색 버튼이 생깁니다.

이 버튼을 꾹 누르면 끝입니다.


---


팀 프로젝트(antigravity)를 진행 중이시라면, 터미널이나 VS Code에서 직접 합치는(Merge) 것은 비추천합니다.

대신 깃허브 웹사이트에서 'Pull Request(PR)' 방식을 사용하는 것이 팀 협업의 표준입니다. 이 방식은 코드를 합치기 전에 동료들이 검토(Code Review)할 수 있는 기회를 줍니다.

팀원들에게 공유할 수 있는 [깃허브 웹사이트에서 안전하게 합치는 법] 가이드입니다.

🌐 [팀 가이드] 깃허브 웹사이트에서 Merge 하기 (Pull Request)
우리는 내 컴퓨터에서 몰래 합치지 않습니다. **"나 다 했으니 합쳐줘!"**라고 공개적으로 요청하고 승인받는 방식을 씁니다.

1단계: 깃허브에 요청서 보내기 (PR 생성)
VS Code에서 Publish Branch(업로드)를 마쳤다면, 깃허브 웹사이트의 우리 프로젝트 페이지로 접속합니다.

상단에 노란색 박스로 "Compare & pull request" 버튼이 생긴 것을 볼 수 있습니다. 클릭하세요.

(버튼이 안 보인다면 'Pull requests' 탭 → 'New pull request'를 누르고 내 브랜치를 선택하세요.)

제목과 내용 작성: "로그인 버튼 디자인 수정했습니다" 처럼 무엇을 했는지 팀원들이 알기 쉽게 적습니다.

Create pull request 녹색 버튼 클릭!

이제 "내 코드를 합쳐주세요"라는 요청서(PR)가 등록되었습니다.

2단계: 팀원 검토 및 합치기 (Review & Merge)
이 단계는 보통 **팀장님(본인)**이나 다른 동료가 수행합니다. (자기가 올리고 자기가 합치는 경우도 있지만, 팀원끼리 서로 봐주는 게 좋습니다.)

등록된 PR 페이지에서 Files changed 탭을 눌러 코드를 확인합니다. (이상한 점이 없나 봅니다.)

문제가 없다면 다시 Conversation 탭으로 돌아와서...

하단의 큰 녹색 버튼 Merge pull request를 누릅니다.

Confirm merge를 눌러 확정합니다.

축하합니다! 이제 feature 브랜치의 내용이 main에 완전히 합쳐졌습니다.

보라색 Delete branch 버튼이 나오면 눌러서 깃허브상의 브랜치를 정리해 줍니다.

⚠️ [매우 중요] 합친 후, 내 컴퓨터 동기화하기 (Git Pull)
웹사이트에서 합쳤으니, 깃허브의 main은 최신 상태가 되었습니다. 하지만 내 컴퓨터(로컬)의 main은 아직 옛날 상태입니다.

모든 팀원은 작업 시작 전, 혹은 Merge 직후에 반드시 아래 과정을 거쳐야 합니다. (이걸 안 하면 나중에 충돌 납니다!)

VS Code 소스 제어 탭에서:

왼쪽 하단 브랜치를 클릭해 **main**으로 이동(Switch)합니다.

소스 제어 탭의 ... 메뉴(혹은 동기화 아이콘)를 누르고 **Pull(당겨오기)**을 클릭합니다.

또는 터미널에서: git pull origin main

이제 내 컴퓨터도 최신 상태가 되었습니다! 다시 새로운 작업을 시작(create branch)하면 됩니다.

🔄 팀장님을 위한 3줄 요약 (팀 전파용)
팀원들에게 이렇게 딱 정해 주시면 깔끔합니다.

작업 끝내면: VS Code에서 Publish 누르고, 깃허브 가서 녹색 버튼(PR) 만들기.

합칠 때는: 깃허브 웹사이트에서 Merge 버튼 누르기.

다시 작업할 땐: VS Code에서 main으로 가서 Pull 받고 나서, 새 브랜치 만들기.

여기까지 하면 기본적인 협업 사이클이 완성됩니다! 혹시 Merge 버튼이 회색으로 비활성화되어 있거나, **"Conflict(충돌)"**라는 무시무시한 빨간 글씨가 떴나요? (두 사람이 같은 파일의 같은 줄을 수정했을 때 발생합니다.) 이것도 해결 방법이 필요하신가요?

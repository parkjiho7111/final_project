정책 카드 더보기 (AJAX 리셋 방식)

목표
페이지 리로드 없이 AJAX로 새 데이터를 불러와서 즉시 화면을 리셋. script.js의 CardSwiper 클래스를 재사용.

수정파일
main_page.py
main.html

제약사항
static/script.js는 수정 불가 (read-only)
script.js의 CardSwiper 클래스와 createCardHTML 함수를 재사용
구현 계획
Step 1: Backend 수정 (이미 완료)
routers/main_page.py의 read_main 함수에 exclude_ids 파라미터가 이미 추가되어 있음.

Step 2: Backend API 엔드포인트 확인
/api/main/more-cards 엔드포인트가 이미 존재하지만 GET 방식. POST 방식으로 변경하거나, GET 방식으로 사용 가능한지 확인 필요.

옵션 1: 기존 GET 엔드포인트 사용 (/api/main/more-cards?region=...&exclude_ids=...)

옵션 2: POST 엔드포인트 새로 추가 또는 기존 것 수정

현재 코드를 보면 /api/main/more-cards가 GET 방식으로 존재. 이를 사용하거나 POST 방식으로 변경.

Step 3: Frontend 수정 (templates/main.html)
3.1 loadMoreCards() 함수 수정 (페이지 리로드 → AJAX)
로직:

현재 화면에 로드된 카드들의 ID 수집 (window.tinderData)
sessionStorage에서 seen_policy_ids 가져오기
중복 제거하여 합치기
버튼 상태 변경 ("새로운 정책 찾는 중...", 비활성화)
AJAX로 새 데이터 가져오기 (페이지 리로드 없이)
받은 데이터로 window.tinderData 업데이트
#tinder-list 컨테이너 완전히 비우기
새로운 CardSwiper 인스턴스 생성하여 카드 다시 그리기
.no-more-cards 요소 숨기기
데이터 없으면 alert 및 버튼 비활성화
